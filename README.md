# 多病历纵向罕见病筛查鉴别MCP工具

## 项目简介

本工具是基于HTTP API + 大模型的罕见病筛查辅助参考系统，采用RAG循证架构，结合中英文医学文献、诊疗指南、药品说明书进行多维度推理分析。工具输出仅为排查思路参考，不构成医学诊断。

### 核心特性

- **多病历纵向分析**: 支持多条带时间顺序的病历输入，自动解析病程演变
- **RAG循证架构**: 充分调用4类官方证据检索接口（中英文文献、诊疗指南、药品说明书），不依赖大模型凭空推理
- **大模型协同**: 基于StepFun阶跃星辰大模型（step-3.5-flash）进行文本理解、信息抽取、推理分析
- **MCP异步任务**: 提交任务获得jobId，支持轮询查询进度，符合MCP扩展工具设计规范
- **合规安全**: 全局医疗合规声明，严禁确定性诊断输出，不持久化用户数据

### 技术架构

```
输入: 多条时序病历文本 + 家族史 + 已排除疾病 + 异常指标
  ↓
步骤1: LLM结构化解析 → JSON（时间线、症状、指标趋势、家族史等）
  ↓
步骤2: LLM生成检索Query → 批量调用4类evidence接口 → 汇总证据
  ↓
步骤3: LLM综合推理（结构化病历 + 证据）→ 三级可疑等级分类
  ↓
步骤4: LLM格式化输出 → Markdown完整筛查报告 + 参考文献列表
  ↓
输出: 罕见病排查方向参考报告（附合规声明）
```

## 使用说明

### 输入说明

1. **多条时序病历文本**: 按时间顺序粘贴多条病历，每条标注时间（如"【2024年3月 首次就诊】"）
2. **家族史**: 描述家族中相关疾病史
3. **已排除疾病**: 列出已经排除的常见病
4. **持续性异常症状**: 描述持续存在的异常表现
5. **异常检验指标**: 汇总异常检验指标及变化趋势

### 内置示例

- **示例1**: 疑似法布雷病（Fabry Disease）- 多系统受累，肢端疼痛，蛋白尿，角膜涡状混浊
- **示例2**: 疑似肝豆状核变性（Wilson Disease）- 肝功能异常，神经症状，K-F环

点击界面上的"填充示例病历"按钮可一键加载。

### 输出说明

- **筛查报告**: Markdown格式，包含病程概述、核心特征、三级可疑等级分析、下一步建议
- **参考文献列表**: 所有引用的中英文文献、指南、药品说明书

## 合规说明

1. **严禁确诊**: 所有输出均为排查方向参考，不构成医学诊断
2. **隐私保护**: 不持久存储用户输入，用完即销毁
3. **禁止真实数据**: 严禁录入真实患者隐私病历，请使用模拟/脱敏数据
4. **全局声明**: 页面顶部和报告末尾固定展示医疗合规免责声明

## 技术细节

### API调用

- **KNOWS API**: 异步任务创建/查询 + 4类证据检索接口
- **大模型API**: StepFun阶跃星辰，OpenAI兼容格式调用
- **认证**: Bearer Token认证

### 异常处理

- 所有HTTP请求含超时（60-120秒）
- 大模型调用含重试（最多3次）
- 证据检索失败时降级为无证据模式
- 全程日志记录便于调试

### 合规措辞约束

LLM输出强制清洗，禁用词汇：
- ❌ 确诊
- ❌ 患病  
- ❌ 就是XX病

统一替换为：✅ 该罕见病为重点排查可疑方向

- **完整RAG循证架构**：不是纯LLM凭空推理，而是“LLM理解→证据检索→综合推理”三步闭环，4类evidence接口全部调用（中英文文献、诊疗指南、药品说明书）
- **代码模块化清晰**：config / knows_api / llm_engine / pipeline / mcp_task / app 六层解耦，职责单一，可直接阅读调试

- **全链路异常处理**：HTTP超时、LLM重试、证据检索降级、合规措辞清洗，生产级健壮性

- **真实痛点**：罕见病平均确诊需5-7年、辗转4-8个科室，本工具从多病历纵向视角切入，辅助医生发现跨系统、跨时间的罕见病线索
- **目标用户明确**：面向临床医生（非患者），定位为“排查思路参考”而非诊断工具

  - 不持久化用户数据，禁止录入真实病历
- **10套模拟病历覆盖多病种**：法布雷病、肝豆状核变性、戈谢病、MPS I、SMA、PKU、DMD、血友病A、成骨不全、MELAS，涵盖溶酶体贮积、代谢、神经肌肉、凝血、遗传等多类罕见病
